이 작업은 **로그인이나 서버 없이 즉시 사용할 수 있는 휘발성 메모 앱**을 목표로 진행한 프로젝트입니다.  
브라우저 환경만으로 어디서든 빠르게 메모를 남기고 정리할 수 있도록 설계했으며,  
단순한 기능 구현을 넘어 **컴포넌트 주도 개발(CDD)** 방법론을 실제 서비스 구조에 적용하는 데 중점을 두었습니다.

---

## 배경

Memo는 “오래 남길 필요 없는 기록”에 집중한 도구입니다.  
복잡한 캘린더나 계정 기반 서비스 대신, **즉시 기록하고 필요 없으면 바로 삭제**하는 사용 패턴을 전제로 설계되었습니다.

또한 UI를 기능 단위가 아닌 **컴포넌트 단위로 사고하고 관리**하기 위해 Storybook을 중심으로 한 개발 흐름을 구축했습니다.  
디자인은 최대한 단순하게 유지하되, 프론트엔드 설계에서 중요한 요소들을 모두 담는 것을 목표로 했습니다.

---

## 해결한 문제

- **컴포넌트 독립성 확보**  
  Storybook을 도입해 비즈니스 로직과 UI를 분리하고,  
  메모 아이템·입력창·버튼 등을 독립 환경에서 개발·검증했습니다.

- **정적 타입 기반 안정성**  
  TypeScript로 메모 데이터 모델과 Props 인터페이스를 명확히 정의하여  
  상태 전달 과정에서 발생할 수 있는 오류를 컴파일 단계에서 차단했습니다.

- **UI 일관성 유지**  
  Empty, Filled, Long Text 등 다양한 상태를 Storybook에서 사전 검증해  
  엣지 케이스에서도 깨지지 않는 레이아웃을 구현했습니다.

- **오프라인 대응**  
  PWA 적용을 통해 네트워크 연결이 없는 환경에서도  
  메모 조회·작성이 가능한 사용자 경험을 제공했습니다.

---

## 기술적 선택 근거

- **TypeScript**  
  상태와 데이터 흐름이 늘어나는 상황에서도 코드 가독성과 안정성을 유지하기 위해 채택했습니다.

- **Storybook**  
  UI 컴포넌트를 문서화하고, 페이지 구현 이전에 시각적으로 검증할 수 있는 환경을 만들기 위해 사용했습니다.

- **PWA**  
  웹 기반 서비스이지만 네이티브 앱에 가까운 경험을 제공하고자 했습니다.  
  Service Worker 기반 캐싱 전략으로 로딩 속도와 접근성을 개선했습니다.

- **Vanilla HTML & CSS**  
  외부 UI 프레임워크 의존성을 줄이고,  
  컴포넌트 구조와 스타일링 설계 자체에 집중하기 위해 선택했습니다.

- **LocalStorage API**  
  서버 없이 즉각적인 데이터 반응성을 확보하기 위한 클라이언트 사이드 저장소로 활용했습니다.

## Memo: 휘발성 플래너

**Memo**는 단기적인 메모와 일정 관리를 위한 **휘발성 플래너**입니다.  
복잡한 기능 없이 빠르게 기록하고, 필요 없으면 즉시 삭제할 수 있도록 설계되었습니다.

---

## 주요 기능

### 빠른 메모 작성
- 단순 텍스트 메모
- 할 일(Todo) 체크리스트
- 간단한 노트 형식

### 즉각적인 관리
- 메모 추가 및 삭제
- 리스트 항목 간편 편집

### 데이터 관리
- 기기 간 데이터 동기화
- Import / Export를 통한 백업 및 복원

### PWA & 앱 지원
- PWA 설치 지원
- Android / iOS 앱 다운로드 링크 제공

### UI 설계
- 빠른 액세스를 위한 플로팅 버튼
- 즉시 사용 가능한 간결한 인터페이스
